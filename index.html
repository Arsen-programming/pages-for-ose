Cегодня мы расскажем про git workflow

Что такое? Это набор практик, которые помогают нам работать с командой через GIT , например ты работаешь в компании и вам дали проект, твой товарищ делает одну часть ты другую и таким образом можно их соединить и сделать одну страницу

Как работает? Создание веток, работа с коммитами, слияние веток, решение конфликтов, развертывание

Для чего нужен?Он помогает комнадам работать над различными проектами одновременно, позволяет отслеживать изменений, может проводить тесты и проверки перед слиянием

GIT rebase

Что это? Это команда в Git, используется для изменение ветки, что нам позволяет переписывать историю коммитов

Отличия от git merge: это вообщем два способа изменений из одной ветки в другую, но они делают по разному, git merge:сохранение истории, слияние более понятнее для новичков, git rebase:он переписывает историю перемещая коммиты из одной ветку на вершину ветки, он изменяет существующие коммиты но это опасно для публичных веток

Git fetch

Что это?это комманда в Git, используется для изменение из удаленного репозитория

Отличия от git pull: это команды в Git, которые используется для работы с удалениями репозиториями, git fetch:загружает новые коммиты и обновления из удаленного репозитория, git pull: фактически выполняет git fetch, а затем автоматически сливает изменения в вашу текущую ветку

GitHub Actions

Что это?Это встроенная команда GitHub, позволяет автоматизировать рабочие процессы разработки.С ее помощью вы можете выполнять команды при определенных событиях в репозитории, таких как коммиты создание pull request или выпуск релизов

Как работает?Вы настраиваете рабочий процесс, который срабатывает на определенные события, например как push,release и другие.Рабочий процесс состоит из одной или нескольких задач.После выполнения рабочего процесса GitHub предоставляет логи, где вы можете видеть результаты каждого шага
